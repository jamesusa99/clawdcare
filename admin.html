Access rules & environment
  • Local (non-production): if ADMIN_EMAILS is empty, any signed-in user can open this page. Set ADMIN_EMAILS=you@email.com to restrict.
  • Production: your email must be in ADMIN_EMAILS or you need an admin role in the database.
  • Tighter local mode: STRICT_ADMIN=1 or ADMIN_DEV_OPEN=0 disables the empty-ADMIN_EMAILS shortcut.

How to get access

  1. Set ADMIN_EMAILS in .env to your email (exact address you use to sign in), e.g. ADMIN_EMAILS=you@example.com, then restart the server and refresh this page.
  2. Or add "admin" to the roles array for your user in data/users.json (file mode) or in Supabase profiles.roles.
  3. Local: leave ADMIN_EMAILS empty (default) so any signed-in user can access admin, or set ADMIN_EMAILS=your@email and restart.
